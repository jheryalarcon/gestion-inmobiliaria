# Historia de Usuario: Visualización de Agentes Registrados

La información detallada en la historia se puede hallar en el anexo correspondiente. Para el desarrollo de esta funcionalidad, se creó la página `PanelAgentes.jsx` utilizando React y Tailwind CSS para permitir a administradores ver una lista completa de todos los agentes inmobiliarios con información detallada, estadísticas de actividad, y opciones para gestionarlos. La interfaz incluye filtros de búsqueda, paginación, y visualización de estadísticas (propiedades, clientes, negociaciones) por agente.

La interfaz fue realizada utilizando componentes de React con hooks `useState` y `useEffect`, y la librería Axios para peticiones HTTP. El componente verifica primero que el usuario esté autenticado y sea administrador antes de mostrar el contenido. El componente carga los agentes al montarse mediante GET `/api/agentes` con parámetros de consulta (page, limit, search, estado) y el token JWT. El componente incluye filtros de búsqueda: campo de texto para buscar por nombre o email, selector de estado (activo, inactivo, todos), y botón para limpiar filtros. Los filtros se aplican automáticamente al cambiar mediante `useEffect` que recarga los agentes cuando cambian `search` o `estado`, reseteando a la primera página.

La tabla muestra columnas: Agente (nombre, email, avatar con inicial), Teléfono, Propiedades (badge con contador), Clientes (badge con contador), Negociaciones (badge con contador), Estado (badge activo/inactivo), Fecha registro (formateada), y Acciones (botón editar, botón desactivar/reactivar según estado). Cada agente muestra estadísticas mediante badges de colores. El componente incluye paginación completa con controles para navegar entre páginas, información de resultados mostrados, y botones de página numerados. El componente incluye modales de confirmación para desactivar y reactivar agentes, y manejo de estados de carga y error.

En el backend se utilizó Node.js con Express.js y Prisma como ORM. La función `obtenerAgentes` en `agentes.controller.js` recibe parámetros de consulta: `page` (default 1), `limit` (default 10), `search` (texto de búsqueda), y `estado` (activo, inactivo, todos, default 'activo'). La función verifica primero que solo administradores puedan ver la lista (retorna 403 si no es admin), construye un objeto `whereClause` con filtro de rol 'agente', agrega filtro de estado según el parámetro, agrega filtro de búsqueda si hay texto mediante `OR` con búsqueda insensible a mayúsculas/minúsculas en nombre, email, y teléfono, calcula el offset para paginación, y obtiene los agentes con `findMany` incluyendo `_count` para propiedades, clientes, y negociaciones. La función también obtiene el total de agentes con `count` para calcular la paginación, calcula el total de páginas, y retorna los agentes con información de paginación.

La ruta GET `/api/agentes` fue configurada en `agentes.routes.js` con middlewares aplicados a todas las rutas mediante `router.use`: `verificarToken` (autenticación) y `esAdmin` (autorización: solo administradores pueden ver la lista), registrada en `index.js` bajo el prefijo `/api/agentes`. El sistema incluye estadísticas agregadas mediante `_count` en Prisma, calculándose en una sola consulta y evitando consultas adicionales.
