# Historia de Usuario: Guardar Propiedad como Favorita

La información detallada en la historia se puede hallar en el anexo correspondiente. Para el desarrollo de esta funcionalidad, se implementó la capacidad de que usuarios con rol `cliente` marquen propiedades de su interés para acceder rápidamente a ellas posteriormente. Se creó el componente `FavoritoIcon.jsx` utilizando React que muestra un icono de corazón clickeable, y la página `MisFavoritos.jsx` que lista todas las propiedades favoritas del usuario.

La interfaz fue realizada utilizando componentes de React y Tailwind CSS. El componente `FavoritoIcon.jsx` muestra un icono de corazón que cambia de color según si la propiedad está en favoritos (rojo si está, gris si no está). El componente verifica si el usuario está autenticado y tiene rol `cliente` antes de permitir la acción, muestra un icono gris y redirige a login si no está autenticado, y muestra un mensaje de error si el usuario no es cliente. Al hacer clic, envía una petición POST para agregar o DELETE para quitar según el estado actual, muestra notificaciones toast de éxito o error mediante Sonner, actualiza el estado local, y ejecuta un callback `onToggle` para notificar al componente padre. La página `MisFavoritos.jsx` carga los favoritos del usuario mediante GET `/api/favoritos` con token de autenticación, extrae las propiedades de los favoritos, muestra un grid responsivo usando `CardPropiedadPublica`, incluye ordenamiento por fecha, precio o título (ascendente/descendente), muestra estadísticas del número de favoritos, y maneja el caso cuando no hay favoritos mostrando un mensaje motivacional.

En el backend se utilizó Node.js con Express.js y Prisma como ORM. El modelo `Favorito` fue definido en `schema.prisma` con campos `id`, `usuarioId`, `propiedadId`, y `createdAt`, estableciendo relaciones con `Usuario` y `Propiedad`, y tiene un constraint único `@@unique([usuarioId, propiedadId])` que previene que un usuario agregue la misma propiedad dos veces. La función `addFavorite` en `favorite.controller.js` recibe el `propiedadId` desde el cuerpo de la petición y el `usuarioId` desde el usuario autenticado, valida que la propiedad exista, verifica si ya está en favoritos usando el constraint único, y crea el registro de favorito si no existe. La función `removeFavorite` elimina el favorito usando la clave compuesta. La función `getFavorites` retorna todos los favoritos del usuario incluyendo información completa de las propiedades mediante `include`.

Las rutas POST `/api/favoritos` (agregar), DELETE `/api/favoritos` (quitar), y GET `/api/favoritos` (listar) fueron configuradas en `favorite.routes.js` con middleware `verificarToken` para requerir autenticación, registradas en `index.js` bajo el prefijo `/api/favoritos`. El sistema valida que solo usuarios con rol `cliente` puedan gestionar favoritos.
