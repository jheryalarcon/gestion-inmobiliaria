# Historia de Usuario 6: Edición de Propiedades

La información detallada en la historia se puede hallar en el anexo correspondiente. Para el desarrollo de la historia 6, se creó la página `EditarPropiedad.jsx` utilizando React y Tailwind CSS para permitir a administradores y agentes modificar propiedades existentes. La interfaz incluye un formulario prellenado con todos los campos de la propiedad y gestión flexible de imágenes (eliminar actuales y agregar nuevas).

La interfaz fue realizada utilizando componentes de React con hooks `useState` y `useEffect`, componentes reutilizables `SelectProvincia.jsx` y `SelectTipoPropiedad.jsx`, y la librería Axios para peticiones HTTP. El componente carga la propiedad al montar mediante GET `/api/propiedades/:id` con token JWT, guarda datos iniciales en `initialDatos` para comparación, y gestiona tres estados de imágenes: `imagenesActuales` (existentes con IDs), `imagenesAEliminar` (IDs a eliminar), y `imagenesNuevas` (archivos nuevos). Incluye protección ante recarga mediante `beforeunload` y modal de confirmación al cancelar si hay cambios.

Al enviar el formulario, la función `handleSubmit` valida todos los campos, crea un `FormData` con datos y archivos, agrega IDs de imágenes a eliminar en `imagenesAEliminar[]`, y realiza una petición PUT a `/api/propiedades/:id` con `Content-Type: multipart/form-data`. Si es exitoso, muestra toast de éxito y redirige al panel de propiedades. El componente verifica permisos en frontend: si es agente y no es propietario, muestra mensaje de acceso denegado.

En el backend se utilizó Node.js con Express.js y Prisma como ORM. La función `obtenerPropiedadPorId` en `propiedad.controller.js` busca la propiedad por ID incluyendo imágenes y datos del agente, verifica permisos según rol (agente solo puede ver sus propias propiedades), y procesa URLs de imágenes. La función `actualizarPropiedad` valida campos obligatorios y opcionales, verifica que la propiedad exista (retorna 404 si no existe), verifica permisos (agente solo puede editar sus propias propiedades, retorna 403 si no), actualiza la propiedad con `prisma.propiedad.update` convirtiendo valores a tipos apropiados, elimina imágenes seleccionadas con `prisma.imagen.deleteMany` usando IDs en `imagenesAEliminar`, y agrega nuevas imágenes con `prisma.imagen.createMany` usando URLs relativas.

El middleware `esPropietarioOAdmin.js` verifica permisos antes de permitir edición: busca la propiedad en la base de datos, verifica que el usuario sea admin o que `propiedad.agenteId === usuario.id`, y retorna 403 si no tiene permisos. La ruta GET `/api/propiedades/:id` fue configurada con middleware `verificarToken`, y la ruta PUT `/api/propiedades/:id` con middlewares `verificarToken`, `esPropietarioOAdmin` (autorización), `upload.array('imagenes', 5)` (procesamiento de nuevas imágenes), y `actualizarPropiedad` (controlador), registradas en `index.js` bajo el prefijo `/api/propiedades`.
