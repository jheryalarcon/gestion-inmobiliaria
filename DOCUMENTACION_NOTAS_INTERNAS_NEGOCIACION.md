# Historia de Usuario: Notas Internas por Negociación

La información detallada en la historia se puede hallar en el anexo correspondiente. Para el desarrollo de esta funcionalidad, se creó el componente `NotasInternas.jsx` utilizando React y Tailwind CSS para permitir a los agentes crear y gestionar notas completamente privadas sobre sus negociaciones. Estas notas son exclusivamente visibles para el agente responsable de la negociación y no pueden ser accedidas por administradores ni otros agentes.

La interfaz fue realizada utilizando componentes de React con hooks `useState` y `useEffect`, y la librería Axios para peticiones HTTP. El componente verifica permisos antes de cargar datos mediante `canViewNotas()` que compara `negociacion.agenteId === usuario.id` y verifica que el usuario sea agente. Si el usuario no puede acceder (administrador u otro agente), muestra un mensaje informativo explicando que las notas son privadas y solo el agente responsable puede verlas. Si el usuario es el agente responsable, carga las notas al montarse mediante GET `/api/notas-internas/:negociacionId` con el token JWT. El componente incluye un formulario para crear nuevas notas que solo se muestra si el usuario es el agente responsable. El formulario incluye un textarea para el contenido con límite de 2000 caracteres, contador de caracteres visible, placeholder con sugerencias de uso, y botones de cancelar y guardar. Al enviar el formulario, se valida que el contenido no esté vacío, se envía una petición POST a `/api/notas-internas/:negociacionId` con `contenido`, y el token JWT. Si la creación es exitosa, muestra un toast de éxito mediante Sonner, limpia el formulario, oculta el formulario, recarga las notas, y ejecuta el callback `onNotaCreada` si está definido. El componente muestra la lista de notas ordenadas cronológicamente (más recientes primero), cada una con icono de candado (🔒), badge "Nota Privada", fecha formateada, contenido completo, y mensaje indicando que es una nota personal no editable. Si no hay notas, muestra un mensaje motivacional con sugerencias de uso. El componente incluye un mensaje de privacidad garantizada en la parte inferior.

En el backend se utilizó Node.js con Express.js y Prisma como ORM. El modelo `NotaInterna` fue definido en `schema.prisma` con campos: `id` (Int autoincrement), `negociacionId` (Int con relación a Negociacion), `agenteId` (Int con relación a Usuario agente), `fecha` (DateTime con default now()), y `contenido` (String con máximo 2000 caracteres). El modelo no incluye campo `activo` ya que las notas internas no se eliminan, manteniendo un historial completo. La función `obtenerNotasInternas` en `notaInterna.controller.js` verifica que la negociación exista y esté activa, verifica permisos estrictos (solo el agente responsable puede ver sus notas mediante `negociacion.agenteId === agenteId`, retorna 403 si no tiene permisos), y obtiene solo las notas del agente responsable ordenadas por fecha descendente usando `prisma.notaInterna.findMany`. La función `crearNotaInterna` valida que el contenido esté presente y no esté vacío (retorna 400 si falta), valida que el contenido no exceda 2000 caracteres (retorna 400 si excede), verifica que la negociación exista y esté activa, verifica permisos estrictos (solo el agente responsable puede crear notas mediante `negociacion.agenteId === agenteId`, retorna 403 si no tiene permisos), verifica que el usuario sea agente (retorna 403 si no es agente, los administradores no pueden crear notas), y crea la nota con `fecha` establecida a la fecha actual usando `prisma.notaInterna.create`.

Las rutas GET `/api/notas-internas/:negociacionId` (obtener) y POST `/api/notas-internas/:negociacionId` (crear) fueron configuradas en `notaInterna.routes.js` con middlewares `verificarToken` (autenticación) y `validarCrearNotaInterna` (validación usando express-validator para validar que `contenido` tenga entre 1 y 2000 caracteres), registradas en `index.js` bajo el prefijo `/api/notas-internas`. El sistema implementa permisos estrictos: solo el agente responsable puede ver y crear notas, los administradores y otros agentes no pueden acceder a ellas.
