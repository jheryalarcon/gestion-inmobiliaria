# Historia de Usuario: Adjuntar Archivos por Negociación

La información detallada en la historia se puede hallar en el anexo correspondiente. Para el desarrollo de esta funcionalidad, se creó el componente `ArchivosAdjuntos.jsx` utilizando React y Tailwind CSS para permitir a los agentes y administradores subir, visualizar y descargar archivos relacionados con una negociación. Los archivos se almacenan en el servidor y se categorizan por tipo (contrato, cédula, otros).

La interfaz fue realizada utilizando componentes de React con hooks `useState` y `useEffect`, y la librería Axios para peticiones HTTP. El componente carga los archivos al montarse mediante GET `/api/archivos-negociacion/negociacion/:negociacionId` con el token JWT. El componente incluye un formulario para subir archivos que solo se muestra si el usuario es agente responsable o admin. El formulario incluye un input de tipo file con aceptación de PDF, JPG y PNG, validación de tipo de archivo en el frontend (solo permite PDF, JPG, PNG), validación de tamaño máximo de 5MB, selector de tipo de archivo (contrato, cédula, otros), y botón para subir que abre un modal de confirmación. Al seleccionar un archivo, se valida el tipo y tamaño antes de permitir continuar. Al hacer clic en subir, se abre un modal de confirmación mostrando nombre, tipo y tamaño del archivo. Si el usuario confirma, se envía una petición POST a `/api/archivos-negociacion/subir` con FormData conteniendo el archivo, `negociacionId`, y `tipo`, y el token JWT. Si la subida es exitosa, muestra un toast de éxito mediante Sonner, limpia el formulario, oculta el formulario, y recarga la lista de archivos. El componente muestra la lista de archivos en una tabla con columnas: Archivo (con icono según tipo), Tipo (badge), Fecha, Tamaño (formateado), y Acción (botón de descargar). Cada archivo tiene un botón de descarga que ejecuta la función `descargarArchivo` que envía una petición GET a `/api/archivos-negociacion/descargar/:archivoId` y descarga el archivo usando Blob API.

En el backend se utilizó Node.js con Express.js y Prisma como ORM. El modelo `ArchivoNegociacion` fue definido en `schema.prisma` con campos: `id` (Int autoincrement), `negociacionId` (Int con relación a Negociacion), `agenteId` (Int con relación a Usuario agente), `nombre_archivo` (String para nombre original), `nombre_guardado` (String para nombre único en el servidor), `tipo` (enum TipoArchivo opcional: contrato, cedula, otros), `fecha_subida` (DateTime con default now()), `url` (String para ruta completa del archivo), `tamano` (Int para tamaño en bytes), y `activo` (Boolean con default true para soft delete). El enum `TipoArchivo` define los tipos: contrato, cedula, otros.

Se configuró `uploadNegociacion` en `config/multer.js` usando multer con almacenamiento en disco en la carpeta `uploads/negociaciones/`, límite de tamaño de 5MB por archivo, y filtro de tipos permitidos (PDF, JPG, PNG) mediante validación de mimetype y extensión. Los archivos se guardan con nombres únicos generados mediante timestamp y número aleatorio. La función `subirArchivo` en `archivoNegociacion.controller.js` recibe el archivo desde `req.file` (procesado por multer), `negociacionId` y `tipo` desde el cuerpo de la petición, y el `agenteId` desde el usuario autenticado. La función verifica que la negociación exista y esté activa, verifica permisos (agente solo puede subir a sus negociaciones mediante `negociacion.agenteId === agenteId`, admin puede subir a cualquier negociación), verifica que se haya seleccionado un archivo, y crea el registro en la base de datos con información del archivo usando `prisma.archivoNegociacion.create`. La función `obtenerArchivos` verifica que la negociación exista y esté activa, verifica permisos (solo agente responsable o admin pueden ver), y obtiene todos los archivos activos ordenados por fecha descendente. La función `descargarArchivo` obtiene el archivo con información de la negociación, verifica permisos (solo agente responsable o admin pueden descargar), verifica que el archivo exista en el sistema de archivos, y envía el archivo para descarga mediante `res.download()`.

Las rutas POST `/api/archivos-negociacion/subir` (con middlewares `verificarToken`, `esAgenteOAdmin`, y `uploadNegociacion.single('archivo')`), GET `/api/archivos-negociacion/negociacion/:negociacionId` (con middlewares `verificarToken` y `esAgenteOAdmin`), y GET `/api/archivos-negociacion/descargar/:archivoId` (con middlewares `verificarToken` y `esAgenteOAdmin`) fueron configuradas en `archivoNegociacion.routes.js`, registradas en `index.js` bajo el prefijo `/api/archivos-negociacion`. El sistema valida el tipo y tamaño de archivo tanto en frontend como backend, y solo permite acceso a los archivos a los agentes responsables y administradores.
