# Historia de Usuario: Clasificación de Propiedades por Estado

La información detallada en la historia se puede hallar en el anexo correspondiente. Para el desarrollo de esta funcionalidad, se implementó la capacidad de cambiar el estado de publicación de propiedades (`disponible`, `vendida`, `arrendada`, `reservada`, `inactiva`) y filtrar propiedades por estado. Se creó el componente `ModalActualizarEstado.jsx` utilizando React y Tailwind CSS que muestra un selector de estados y realiza la actualización mediante petición PATCH.

La interfaz fue realizada utilizando componentes de React, el componente `CardPropiedad.jsx` que muestra un badge de estado con colores diferenciados (verde para disponible, rojo para vendida, azul para arrendada, amarillo para reservada), y un botón que abre el modal solo para usuarios con permisos (admin o agente propietario). El modal valida la selección, envía una petición PATCH a `/api/propiedades/:id/estado` con el token de autenticación, muestra mensajes de éxito o error mediante toasts, y ejecuta un callback para actualizar el estado local sin recargar la página.

En el backend se utilizó Node.js con Express.js y Prisma como ORM. El enum `EstadoPropiedad` fue definido en `schema.prisma` con los valores: `disponible`, `vendida`, `arrendada`, `reservada`, `inactiva`. El campo `estado_publicacion` se añadió al modelo `Propiedad` con valor por defecto `disponible`. La función `actualizarEstadoPropiedad` en `propiedad.controller.js` valida el ID de la propiedad, verifica que el nuevo estado esté en la lista de estados permitidos, comprueba que la propiedad exista, y actualiza el campo `estado_publicacion` con `prisma.propiedad.update`. La función `obtenerPropiedades` filtra propiedades excluyendo las que tienen estado `inactiva`.

La ruta PATCH `/api/propiedades/:id/estado` fue configurada en `propiedad.routes.js` con middlewares `verificarToken` (autenticación) y `esPropietarioOAdmin` (autorización: solo admin o agente propietario pueden cambiar el estado), registrada en `index.js` bajo el prefijo `/api/propiedades`. El componente `FiltroPropiedades.jsx` incluye un selector de estado que permite filtrar propiedades por estado en el panel de administración/agente.
