# Historia de Usuario: Visualización de Clientes Registrados

La información detallada en la historia se puede hallar en el anexo correspondiente. Para el desarrollo de esta funcionalidad, se creó la página `PanelClientes.jsx` utilizando React y Tailwind CSS para permitir a administradores y agentes ver, buscar y filtrar clientes mediante una tabla con paginación. Los administradores ven todos los clientes, mientras que los agentes solo ven sus propios clientes.

La interfaz fue realizada utilizando componentes de React con hooks `useState` y `useEffect`, el componente `SelectTipoCliente.jsx` para filtros, y la librería Axios para peticiones HTTP. El componente incluye una sección de filtros con: búsqueda por texto (nombre, email o teléfono), selector de tipo de cliente, selector de estado (activo, inactivo, todos para admin), y botón para limpiar filtros. La tabla muestra columnas: Cliente (nombre y observaciones), Contacto (email y teléfono), Tipo (badge con tipo de cliente), Agente (solo para admin), Estado (badge verde para activo, rojo para inactivo, con información de quién desactivó), Fecha registro, y Acciones (botones de editar, desactivar/reactivar). La paginación se muestra en la parte inferior con controles para navegar entre páginas, información del rango de resultados, y botones anterior/siguiente. Un `useEffect` detecta cambios en los filtros y recarga automáticamente los clientes.

En el backend se utilizó Node.js con Express.js y Prisma como ORM. La función `obtenerClientes` en `cliente.controller.js` recibe parámetros de query opcionales: `page` (default 1), `limit` (default 10), `search` (búsqueda por nombre, email o teléfono), `tipo_cliente` (filtro por tipo), y `estado` (activo, inactivo, todos). La función construye un objeto `where` que aplica filtros según el rol: si es agente, filtra por `agenteId` igual al ID del usuario; si es admin, no aplica filtro por agente. Además, aplica filtros por estado, búsqueda parcial case-insensitive mediante `OR`, y tipo de cliente. La función usa `skip` y `take` para paginación, ejecuta dos consultas en paralelo (una para obtener clientes y otra para contar el total), y retorna clientes con información del agente y usuario desactivador mediante `include`, junto con un objeto de paginación.

La ruta GET `/api/clientes` fue configurada en `cliente.routes.js` con middlewares `verificarToken` (autenticación) y `esAgenteOAdmin` (autorización: solo agentes y administradores pueden ver clientes), registrada en `index.js` bajo el prefijo `/api/clientes`. El sistema implementa paginación en el servidor usando `skip` y `take` en Prisma, optimizando el rendimiento al no cargar todos los clientes a la vez.
