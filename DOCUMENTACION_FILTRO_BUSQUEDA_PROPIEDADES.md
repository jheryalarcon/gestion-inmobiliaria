# Historia de Usuario: Filtro y Búsqueda de Propiedades

La información detallada en la historia se puede hallar en el anexo correspondiente. Para el desarrollo de esta funcionalidad, se implementó la capacidad de buscar y filtrar propiedades por múltiples criterios: tipo de propiedad, ciudad, rango de precio, número de habitaciones, número de baños, tipo de transacción, y búsqueda por título. Se crearon dos implementaciones diferentes: una para la página pública con filtrado opcional en el servidor, y otra para el panel de administración/agente con filtrado completamente en el cliente.

Para la página pública se creó el componente `FiltrosPropiedades.jsx` utilizando React y Material-UI que incluye una barra de búsqueda por título integrada y múltiples selectores para tipo de propiedad, ciudad, transacción, precio mínimo/máximo, habitaciones y baños. El componente carga dinámicamente las opciones de filtros (ciudades, tipos, transacciones) desde la API al montarse, extrayendo valores únicos de las propiedades recibidas. Los filtros se aplican en el cliente mediante la función `aplicarFiltros` en `Propiedades.jsx` que filtra el array de propiedades localmente según los criterios seleccionados.

Para el panel de administración/agente se creó el componente `FiltroPropiedades.jsx` con Material-UI que incluye búsqueda por título, selectores de estado/tipo/ciudad, y rango de precio con slider. El componente aplica filtros en tiempo real mediante la función `filtrar` en `PanelPropiedades.jsx` que filtra por búsqueda parcial case-insensitive, igualdad exacta para estado y tipo, búsqueda parcial para ciudad, y comparaciones numéricas para precio.

En el backend se utilizó Node.js con Express.js y Prisma como ORM. La función `obtenerPropiedadesPublicas` en `propiedad.controller.js` recibe parámetros de query opcionales: `tipo_propiedad`, `ciudad`, `minPrecio`, `maxPrecio`, `nro_habitaciones`, `nro_banos`, `area_construccion_min`, `area_construccion_max`, `transaccion`, `sortBy` (default 'createdAt'), `sortOrder` (default 'desc'), y `limit`. La función construye un objeto `where` que siempre filtra por `estado_publicacion: 'disponible'` y agrega condiciones adicionales según los parámetros usando operadores Prisma (`contains` para ciudad, `gte`/`lte` para rangos). La función `obtenerPropiedades` para el panel no recibe parámetros de filtrado y retorna todas las propiedades según el rol (admin: todas excepto inactivas, agente: solo las suyas excepto inactivas), siendo el filtrado completamente en el cliente.

La ruta GET `/api/propiedades/publicas` fue configurada sin autenticación para acceso público, y la ruta GET `/api/propiedades` con middleware `verificarToken` para el panel, registradas en `index.js` bajo el prefijo `/api/propiedades`.
