# Historia de Usuario: Edición de Agentes

La información detallada en la historia se puede hallar en el anexo correspondiente. Para el desarrollo de esta funcionalidad, se creó la página `EditarAgente.jsx` utilizando React y Tailwind CSS para permitir a administradores modificar la información de agentes inmobiliarios existentes (nombre, email, teléfono). El sistema valida los datos antes de actualizar, verifica que el email sea único excluyendo el agente actual, y previene pérdida de datos mediante protección ante recarga de página.

La interfaz fue realizada utilizando componentes de React con hooks `useState` y `useEffect`, y la librería Axios para peticiones HTTP. El componente carga los datos del agente mediante GET `/api/agentes/:id` al montarse, verifica autenticación y rol admin antes de cargar, y maneja errores de permisos redirigiendo al panel de agentes. El formulario se prellena con los datos del agente cargado (nombre, email, teléfono), y guarda los datos iniciales en `initialDatos.current` para comparación. El formulario incluye validación en tiempo real que verifica campos obligatorios, formato de email, formato de teléfono, y muestra mensajes de error específicos por campo. El componente incluye protección ante recarga de página mediante `beforeunload` y modal de confirmación al cancelar si hay cambios sin guardar.

Al enviar el formulario, se valida completamente, se preparan los datos (campos trim), y se envía una petición PUT a `/api/agentes/:id` con los datos actualizados y el token JWT. Si la actualización es exitosa, muestra un toast de éxito y redirige al panel de agentes.

En el backend se utilizó Node.js con Express.js y Prisma como ORM. La función `obtenerAgentePorId` en `agentes.controller.js` verifica primero que solo administradores puedan ver detalles (retorna 403 si no es admin), busca el agente por ID y rol 'agente', incluye información relacionada (propiedades, clientes, negociaciones), y retorna el agente con toda la información. La función `actualizarAgente` verifica primero que solo administradores puedan actualizar (retorna 403 si no es admin), valida campos obligatorios, valida formato de email y teléfono mediante expresiones regulares, verifica que el agente exista (retorna 404 si no existe), verifica que el email no esté duplicado excluyendo el agente actual mediante consulta con `id: { not: parseInt(id) }` (retorna 400 si ya existe), y actualiza el agente con los nuevos datos usando `prisma.usuario.update`.

Las rutas GET `/api/agentes/:id` (obtener) y PUT `/api/agentes/:id` (actualizar) fueron configuradas en `agentes.routes.js` con middlewares aplicados a todas las rutas mediante `router.use`: `verificarToken` (autenticación) y `esAdmin` (autorización: solo administradores pueden ver y editar agentes), registradas en `index.js` bajo el prefijo `/api/agentes`. El sistema solo permite editar nombre, email y teléfono; el rol, contraseña y estado activo se gestionan por separado.
