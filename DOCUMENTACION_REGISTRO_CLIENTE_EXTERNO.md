# Historia de Usuario 3: Registro de Cuenta (Cliente Externo)

La información detallada en la historia se puede hallar en el anexo correspondiente. Para el desarrollo de la historia 3, se creó la página `Registro.jsx` utilizando React y Tailwind CSS para permitir a usuarios no autenticados crear una cuenta con rol de cliente. La interfaz incluye campos para nombre completo, email, teléfono opcional, y contraseña con validación en tiempo real.

La interfaz fue realizada utilizando componentes de React con hooks `useState` y `useEffect`, la librería Axios para peticiones HTTP, y `jwt-decode` para decodificar tokens. El componente incluye validación en tiempo real mediante expresiones regulares para email y teléfono (10 dígitos si se proporciona), y validación de contraseña con mínimo 6 caracteres. El componente verifica al montar si existe un token válido y redirige automáticamente si el usuario ya está autenticado.

Al enviar el formulario, la función `handleRegistro` valida los campos, normaliza el email (trim y lowercase), y realiza una petición POST a `/api/auth/register` mediante Axios. Si el registro es exitoso, almacena el token JWT y datos del usuario en `localStorage`, dispara el evento `authChange` para actualizar componentes, muestra un toast de éxito mediante Sonner, y redirige a la página principal después de 2 segundos.

En el backend se utilizó Node.js con Express.js y Prisma como ORM para PostgreSQL. El modelo `Usuario` fue definido en `schema.prisma` con campos `name`, `email` (único), `password`, `telefono` (opcional), `rol` (enum), y `activo` (Boolean, default true). La función `register` en `auth.controller.js` valida campos obligatorios, normaliza el email mediante `trim().toLowerCase()`, verifica email único usando `prisma.usuario.findUnique`, encripta la contraseña con `bcrypt.hash(password, 10)` usando 10 salt rounds, crea el usuario con `prisma.usuario.create` asignando siempre rol 'cliente' para registros externos, genera un token JWT con `jsonwebtoken` incluyendo `id` y `rol` con expiración de 1 día, y retorna el token junto con datos del usuario (sin contraseña) en respuesta 201.

La ruta POST `/api/auth/register` fue configurada en `auth.routes.js` sin middlewares de autenticación por ser un endpoint público, y registrada en `index.js` bajo el prefijo `/api/auth`. El sistema implementa autenticación automática después del registro, eliminando la necesidad de iniciar sesión por separado.
