# Historia de Usuario 5: Visualización de Propiedades Registradas

La información detallada en la historia se puede hallar en el anexo correspondiente. Para el desarrollo de la historia 5, se creó la página `PanelPropiedades.jsx` utilizando React y Tailwind CSS para permitir a administradores y agentes visualizar las propiedades que gestionan. La interfaz muestra las propiedades en un grid con tarjetas que incluyen imagen, título, tipo, ciudad, precio, estado con colores, y botones de acción según permisos.

La interfaz fue realizada utilizando componentes de React con hooks `useState` y `useEffect`, el componente `FiltroPropiedades.jsx` con Material-UI para filtros (búsqueda por título, selectores de estado/tipo/ciudad, rango de precio con slider), y el componente `CardPropiedad.jsx` para mostrar cada propiedad. El componente carga propiedades al montar mediante GET `/api/propiedades` con token JWT, aplica filtrado completamente en el cliente mediante la función `filtrar` que filtra por búsqueda, estado, tipo, ciudad, y rango de precio, y actualiza propiedades localmente mediante `actualizarPropiedadLocal` sin recargar la página.

En el backend se utilizó Node.js con Express.js y Prisma como ORM. La función `obtenerPropiedades` en `propiedad.controller.js` filtra propiedades según rol: admin ve todas excepto inactivas usando `prisma.propiedad.findMany` con `estado_publicacion: { not: 'inactiva' }`, agente ve solo las suyas usando `agenteId: usuario.id` y excluyendo inactivas, incluye imágenes mediante `include: { imagenes: true }`, ordena por fecha de creación descendente, y procesa URLs de imágenes mediante `imageConfig.procesarImagenes` que convierte URLs relativas a absolutas.

La función `procesarImagenes` en `config/images.js` itera sobre propiedades e imágenes, verifica si la URL es relativa (comienza con `/`) y la convierte a absoluta agregando el dominio del servidor, o la deja sin cambios si ya es absoluta. La ruta GET `/api/propiedades` fue configurada en `propiedad.routes.js` con middleware `verificarToken` (autenticación), registrada en `index.js` bajo el prefijo `/api/propiedades`. El sistema excluye automáticamente propiedades inactivas de la visualización, manteniendo la lista enfocada en propiedades activas.
