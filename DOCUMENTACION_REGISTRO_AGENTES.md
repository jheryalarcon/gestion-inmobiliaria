# Historia de Usuario: Registro de Agentes

La información detallada en la historia se puede hallar en el anexo correspondiente. Para el desarrollo de esta funcionalidad, se creó la página `RegistrarAgente.jsx` utilizando React y Tailwind CSS para permitir a administradores crear nuevas cuentas de agentes inmobiliarios. La interfaz incluye un formulario completo con secciones para datos personales (nombre, email, teléfono) y credenciales de acceso (contraseña, confirmar contraseña).

La interfaz fue realizada utilizando componentes de React con hooks `useState` y `useEffect`, y la librería Axios para peticiones HTTP. El componente verifica primero que el usuario esté autenticado y sea administrador antes de mostrar el formulario. El formulario incluye validación en tiempo real que verifica campos obligatorios, formato de email mediante expresión regular, formato de teléfono (solo números, 7-15 dígitos), longitud mínima de contraseña (6 caracteres), y coincidencia de contraseñas. El componente incluye protección ante recarga de página mediante `beforeunload` y modal de confirmación al cancelar si hay cambios sin guardar.

Al enviar el formulario, se valida completamente, se preparan los datos (email en minúsculas, campos trim), y se envía una petición POST a `/api/agentes/crear` con el token JWT. Si el registro es exitoso, muestra un toast de éxito con las credenciales del agente creado (email y contraseña) y opción para copiar al portapapeles, limpia el formulario, y redirige al panel de agentes después de 2 segundos.

En el backend se utilizó Node.js con Express.js y Prisma como ORM. La función `crearAgente` en `agentes.controller.js` verifica primero que solo administradores puedan crear agentes (retorna 403 si no es admin), valida campos obligatorios, valida formato de email mediante expresión regular `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`, valida formato de teléfono mediante expresión regular `/^[0-9]{7,15}$/`, valida que la contraseña tenga al menos 6 caracteres, verifica email único usando `prisma.usuario.findUnique`, encripta la contraseña usando `bcrypt.hash(password, 10)` con 10 salt rounds, y crea el usuario con `prisma.usuario.create` asignando `rol: 'agente'` y `activo: true`. La función retorna información del agente creado sin incluir la contraseña.

La ruta POST `/api/agentes/crear` fue configurada en `agentes.routes.js` con middlewares aplicados a todas las rutas mediante `router.use`: `verificarToken` (autenticación) y `esAdmin` (autorización: solo administradores pueden crear agentes), registrada en `index.js` bajo el prefijo `/api/agentes`.
