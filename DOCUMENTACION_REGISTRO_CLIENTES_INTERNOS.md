# Historia de Usuario: Registro de Clientes Internos

La información detallada en la historia se puede hallar en el anexo correspondiente. Para el desarrollo de esta funcionalidad, se creó la página `RegistrarCliente.jsx` utilizando React y Tailwind CSS para permitir a administradores y agentes registrar clientes potenciales con información de contacto, tipo de cliente, y observaciones. La interfaz incluye un formulario completo con secciones para datos personales e información adicional, y selector de agente responsable solo para administradores.

La interfaz fue realizada utilizando componentes de React con hooks `useState` y `useEffect`, el componente reutilizable `SelectTipoCliente.jsx` para selector de tipo de cliente, y la librería Axios para peticiones HTTP. El componente carga la lista de agentes activos si el usuario es admin mediante GET `/api/usuarios/agentes`. El formulario incluye validación en tiempo real que verifica campos obligatorios, formato de email mediante expresión regular, formato de teléfono, y muestra mensajes de error específicos por campo. El componente incluye protección ante recarga de página mediante `beforeunload` y modal de confirmación al cancelar si hay cambios sin guardar.

Al enviar el formulario, se valida completamente, se preparan los datos (eliminando agenteId si no es admin), y se envía una petición POST a `/api/clientes` con el token JWT. Si el registro es exitoso, muestra un toast de éxito mediante Sonner y redirige al panel de clientes. Si hay errores, muestra toasts con los mensajes de error del backend.

En el backend se utilizó Node.js con Express.js y Prisma como ORM. El modelo `Cliente` fue definido en `schema.prisma` con campos obligatorios: `nombre`, `telefono`, `email` (único), `tipo_cliente` (enum TipoCliente), y campos opcionales: `observaciones`, `agenteId` (Int opcional para relación con Usuario agente). El modelo incluye campo `activo` (Boolean, default true) para soft delete, y campos de auditoría `fecha_desactivacion` y `desactivado_por` para registrar quién y cuándo desactivó el cliente. El enum `TipoCliente` define los tipos: comprador, arrendatario, propietario, vendedor, inversionista, consultor.

La función `crearCliente` en `cliente.controller.js` valida campos obligatorios, valida formato de email mediante expresión regular, verifica email único usando `prisma.cliente.findUnique`, valida agente asignado si es admin (debe seleccionar un agente activo), asigna agente automáticamente según rol (admin selecciona agente, agente se auto-asigna), y crea el cliente con `prisma.cliente.create` incluyendo información del agente asignado en la respuesta. La ruta POST `/api/clientes` fue configurada en `cliente.routes.js` con middlewares `verificarToken` (autenticación) y `esAgenteOAdmin` (autorización: solo agentes y administradores pueden registrar clientes), registrada en `index.js` bajo el prefijo `/api/clientes`.
